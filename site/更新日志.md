## 2026-07-15 (v11 — 后端健壮性：未绑 KV 时给出明确错误，不再抛 1101)
- 用户反馈：正式站「服务器断开」，Cloudflare Metrics 显示「脚本例外」49 次
- 根因定位：所有 `/api/*` 函数均抛出 `Worker threw exception (1101)`，几乎都是因为运行时 `env.NEWS_KV` 为 undefined（KV 绑定未生效/未绑定/变量名写错），访问 undefined 的 `.get()` 直接崩溃
- 修复：
  1. 新增 `functions/api/_kv.js` 提供 `json()` 与 `checkKV()` 辅助
  2. 所有接口（visits / comments / guestbook / stats / 新增 health）先检查 `env.NEWS_KV`，缺失时返回 **503 JSON** `{ok:false, error:"KV binding NEWS_KV not found"}`，不再抛 1101
  3. 统一用 `new Response(JSON.stringify(...))` 替代 `Response.json()`，避免兼容性疑虑
  4. 新增 `/api/health` 供前端探测，返回 `{ok:true, kv:true, time:...}`
  5. `store.js` 的 `Store.ping()` 改为调用 `/api/health` 并返回 `{ok, msg}`；`dashboard.html` 根据返回信息显示具体原因
- 预览（纯静态，仍会显示未连接）：https://a094ff9cecb8463da18d17fd3bc0f70c.app.codebuddy.work
- 正式站生效：双击 `push-to-github.bat` 推送 → Cloudflare 重新部署。若仍提示 KV 未找到，请到项目 **Settings → Bindings → KV namespace**，确认变量名是 `NEWS_KV`，且选中了命名空间，然后点「重试部署」
